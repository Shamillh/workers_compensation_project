# workers_compensation_project
Цель проекта — разработать модель машинного обучения для предсказания итоговой стоимости страхового возмещения (UltimateIncurredClaimCost) на основе характеристик работника и параметров случая.

# Прогнозирование страховых выплат (Workers' Compensation)

[![Streamlit App](https://static.streamlit.io/badges/streamlit_badge_black_white.svg)](https://your-app-url.com)
[![Python 3.8+](https://img.shields.io/badge/python-3.8+-blue.svg)](https://www.python.org/downloads/)

Приложение для **прогнозирования итоговой стоимости страхового возмещения** по несчастным случаям на производстве на основе характеристик работника, описания заявки и начальной оценки.

## Цель проекта

Обучить и сравнить несколько моделей машинного обучения для регрессии, выбрать лучшую и предоставить удобный интерфейс для предсказания новых случаев. В качестве целевой переменной используется `UltimateIncurredClaimCost` – окончательная стоимость возмещения.

## Структура репозитория
├── analysis_and_model.py # Основной модуль: загрузка данных, обучение, сравнение моделей, предсказания

├── app.py # Точка входа с навигацией по страницам

├── presentation.py # Статическая страница-презентация проекта

├── Get_Dataset_WorkersCompensation.py # Скрипт для сохранения датасета из OpenML в CSV

├── requirements.txt # Зависимости Python

└── README.md # Документация


## Установка и запуск

### 1. Клонировать репозиторий
```bash
git clone https://github.com/your-username/workers-compensation-prediction.git
cd workers-compensation-prediction
```

### 2. Установить зависимости
```bash
pip install -r requirements.txt
```
Для работы XGBoost может потребоваться дополнительная установка:
```bash
pip install xgboost
```

### 3. (Опционально) Предварительно сохранить датасет локально
```bash
python Get_Dataset_WorkersCompensation.py
```
Скрипт загрузит датасет с OpenML и сохранит как Data/workers_compensation.csv (папка создаётся автоматически).
Это ускорит загрузку при следующем запуске приложения.

### 4. Запустить Streamlit-приложение
```bash
streamlit run app.py
```

## Использование
Приложение состоит из двух страниц (боковое меню):

### 1. Анализ и модель
1.1 Загрузка данных – данные подгружаются автоматически (локальный CSV или с OpenML). Выполняется предобработка:

  - Преобразование дат, расчёт задержки отчёта (ReportingDelay)

  - Кодирование категориальных признаков (Gender, MaritalStatus, PartTimeFullTime, ClaimDescription)

1.2 Обучение моделей – после загрузки нажмите кнопку «Обучить модель». Будут обучены 4 модели:

  - Linear Regression

  - Random Forest (n_estimators=50)

  - XGBoost (n_estimators=50)

  - Ridge Regression (alpha=1.0)

1.3 Сравнение – таблица метрик (MAE, RMSE, R²) и график RMSE.

1.4 График предсказаний лучшей модели – точечный график «реальные vs предсказанные».

1.5 Важность признаков – для Random Forest и XGBoost показывается топ-10 признаков.

1.6 Предсказание нового случая – после обучения появляется форма, где можно ввести характеристики и получить прогноз стоимости.

### 2. Презентация
Статическая страница с описанием цели, данных, моделей и результатов (заполняется после обучения).

## Данные
 * Источник: OpenML ID 42876 (Workers' Compensation)

 * Размер: 100 000 записей

 * Признаки (14):

    `Age` – возраст

    `Gender` – пол

    `MaritalStatus` – семейное положение

    `DependentChildren` – количество детей на иждивении

    `DependentsOther` – другие иждивенцы

    `WeeklyPay` – недельная зарплата

    `PartTimeFullTime` – тип занятости

    `HoursWorkedPerWeek` – часов в неделю

    `DaysWorkedPerWeek` – дней в неделю

    `ClaimDescription` – текстовое описание заявки (закодировано)

    `InitialCaseEstimate` – начальная оценка стоимости

    `AccidentMonth` – месяц происшествия

    `AccidentDayOfWeek` – день недели

    `DateTimeOfAccident`, `DateReported` (используются для вычисления `ReportingDelay` и удаляются)

 *  Целевая переменная: `UltimateIncurredClaimCost`

## Примечания
 * При первом запуске данные загружаются с OpenML (около 100 000 записей) – может занять некоторое время.

 * Для ускорения используйте скрипт `Get_Dataset_WorkersCompensation.py`, чтобы сохранить CSV локально.

 * Все категориальные признаки кодируются с помощью `LabelEncoder`. Текстовое поле `ClaimDescription` также преобразуется в числовой код.

 * Модели обучаются на 80% данных, валидация на 20%.
